# Input vs Output Pins Guide

Configuring pin direction in software dictates whether a pin receives electrical data from the outside world or drives voltage out to power external components.

---

## Core Operational Concepts

Every General Purpose Input/Output (GPIO) pin on your SprouT microcontroller must be assigned a functional direction in your control software:

[ External World ] ---> (INPUT PIN)  ---> [ Microcontroller CPU ]
[ External World ] <--- (OUTPUT PIN) <--- [ Microcontroller CPU ]

---

## Input Pins

An **Input Pin** configures the microcontroller to act as a high-impedance voltage meter that monitors external hardware signals without drawing high current.

### Characteristics
* **Function:** Monitors electrical signals generated by external components.
* **Safety Threshold:** Input pins draw minimal current (microamperes) to protect internal logic circuits.
* **Common Use Cases:**
  * Detecting manual triggers (Push Buttons, Capacitive Touch Pads).
  * Reading environmental sensor data (IR, Ultrasonic, Temperature, Gas).

### Pull-Up and Pull-Down Resistors
When an input pin is not connected to a defined voltage source, it enters a "floating" state where ambient electrical noise causes unpredictable logic toggling between HIGH and LOW.
* **Pull-Up Resistor:** Connects the pin internally to VCC (5V) so the default state is HIGH until pulled LOW by a switch.
* **Pull-Down Resistor:** Connects the pin internally to GND (0V) so the default state is LOW until pulled HIGH by a switch.

---

## Output Pins

An **Output Pin** configures the microcontroller to act as a power source or sink, delivering voltage to drive connected hardware.

### Characteristics
* **Function:** Sends electrical control signals or power out to external modules.
* **Current Limits:**
  * Maximum recommended continuous current per pin: **20 mA to 40 mA** (depending on microcontroller core).
  * **Warning:** Exceeding maximum pin current limits will permanently damage the microchip. High-draw devices (motors, relays) require dedicated driver boards.
* **Common Use Cases:**
  * Lighting status LEDs.
  * Triggering alarm buzzers.
  * Sending control pulses to motor drivers and relays.

---

## Quick Configuration Reference

When programming in PictoBlox or Arduino IDE, follow these signal direction guidelines:

* **Configure as INPUT when reading:**
  * Push Button states (`read digital pin`)
  * Sensor feedback (`read analog pin`)
* **Configure as OUTPUT when driving:**
  * LEDs (`set digital pin HIGH/LOW`)
  * Buzzers (`play tone on pin`)
  * Relays and Motor Drivers (`set digital/PWM output`)
